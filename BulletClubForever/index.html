bullet club forever

A game written in JavaScript using the Phaser framework.


Controls: Left (Cursor Left), Right (Cursor Right), Jump (X)

11/10/2014
There are many options for minifying and concatting js files. Currently I'm using a tool called jsmin in combination with a build step in my VS project that does both steps. It looks like this: "type "$(ProjectDir)GameScripts\*.js" | "$(ProjectDir)Tools\jsmin.exe" > "$(ProjectDir)GameScripts\bcf.min.js"". It works well, as long as I have used semicolons properly across all files. I was recently made aware of Google's Javascript standards and the part about when to use semicolons and the difference between function declarations and function expressions explained why my concatted and minified file wasn't valid javascript at first. So now I have a minified file that contains all my game code. The index.html page uses this, whereas the debug.html page uses the originals for debugging purposes. There may well be a nicer way of doing this, but it will serve for now.

I'm going to start trying to document my progress here in the hope that this will somehow mean that I don't abandon this like I almost certainly will. So far I've got a randomised level map which is infinitely less clever than Spelunky's but I guess inifinitely less clever than perfection isn't that bad. The map is created by this class which also has the capability of adding to itself as the player progresses through the level. I mean to make this a bit clevererer by having the array recycle itself instead of just getting bigger and bigger. The game does have the word forever in the title after all.

Phaser draws sprites/groups in the order that they're added to the game, so I managed to make the grass appear in front of the player, but I think I'm going to abaondon clever draw orders for now and concentrate on getting the basics in place. In case you hadn't noticed, it's not a game yet rather than a highly complex way of drawing pictures on a web page that took ages.
